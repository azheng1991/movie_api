Documentation

Brief: To build the server­side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.


1. Return a list of ALL movies to the users
  • URL: /movies
  • method: GET
  • Request data format: none
  • Response data body format: A JSON object holding data about all movies

2. Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user
  • URL: /movies/[Title]
  • method: GET
  • Request data format: none
  • Request data body format: A JSON object holding data about a single movie, containing an ID, title, and other properties.

3. Return data about a genres (description) by movie title
  • URL: movies/Genre/[Title]
  • method: GET
  • Request data format: none
  • Request data body format: A JSON object holding data about a genre (descriptio) by name or title.

4. Return data about a director (bio, birth year, death year) by name
  • URL: /movies/Director/[Name]
  • method: GET
  • Request data format: none
  • Request data body format: A JSON object holding data about a single director, containing name, birth year, death year.

5. Allow new users to register
  • URL: /users
  • method: POST
  • request data format: A JSON object holding data about new user in the following format: username: pw: email: birthday: favorite movie:
  • Response data body format: A JSON object holding the data above in the same format plus a unique identifier

6. Allow users to update their user info (username, password, email, date of birth)
  • URL: /users/[Username]
  • method: PUT
  • Request data format: A JSON object hodling data about user structured as: username: pw: email: birthday: favorite movie:
  • Response data format: A message stating user info has been successfully updated

7. Allow users to add a movie to their list of favorites
  • URL:/users/:Username/Movies/:MovieID
  • method: POST
  • Request data format: None
  • Response data format: A message stating whether or not the movie has been successfully added.

8. Allow users to remove a movie from their list of favorites
  • URL: /users/:Username/Movies/:MovieID
  • method: DELETE
  • Request data format: none
  • Response data format: A message stating whether or not the action has been completed.

9. Allow existing users to deregister (remove a user by Username or id)
  • URL: /users/[Username]
  • method: DELETE
  • Request data format: None
  • Response data format: A message stating whether or not the action was successful.
